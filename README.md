# 学習塾競合調査マルチエージェントシステム

Claude Code のサブエージェント（Agent）・Skill機能を使い、指定したエリアで展開する複数の総合学習塾（小学生・中学生・高校生を対象とし、大学受験特化の予備校とは異なる）の競合比較を調査し、統合レポート（Markdown）とプレゼン用スライド（pptx）を自動生成するマルチエージェントシステムです。

## 想定読者・用途

- 教育業界への転職活動における企業研究資料・面接対策の材料
- 生成AI・業務自動化スキルを示すポートフォリオ

## アーキテクチャ

| 役割 | 定義ファイル | 担当業務 |
|------|------------|---------|
| PM（メインエージェント） | `CLAUDE.md` | タスク分解・対象エリア/学習塾リストの確定・進捗管理・最終統合 |
| リサーチャー | `.claude/agents/researcher.md` | 学習塾ごとの調査（対象社数ぶん並列実行） |
| プランナー | `.claude/agents/planner.md` | 調査データをもとにレポート構成案を作成 |
| ライター | `.claude/agents/writer.md` | レポート執筆・`juku-report-format` Skillでの整形 |
| レビュアー | `.claude/agents/reviewer.md` | 成果物の品質チェック・合否判定 |
| スライド生成 | `.claude/agents/slide-builder.md` | `md-to-slides` Skillでのpptx生成 |

処理フロー：

```
[researcher × N（学習塾ごとに並列）]
        ↓
    planner（構成案作成）
        ↓
     writer（執筆・整形）
        ↓
    reviewer（品質チェック）
        ↓
  slide-builder（スライド生成）
```

## 特徴：調査対象は都度プロンプトで指定可能

対象エリア・学習塾の数・塾名・対象学年層は固定値ではなく、ユーザーがプロンプトで指定します。例：

```
九州エリアで展開する総合学習塾のうち主要な3〜4社について、
小中高全学年を対象に、コース展開・料金体系・合格実績・DX施策の観点から
競合比較調査をしてください。塾名の指定はないので、researcherに候補を探索させてください。
```

対象学習塾の数が変われば、PMが起動するresearcherサブエージェントの数もそれに応じて増減します。
なお、全国チェーンの大手予備校（河合塾・東進・駿台予備校等）はデフォルトでは対象外です（地域展開の総合学習塾を対象とするシステムのため）。

## 出力構成

| 種別 | 出力先 |
|------|--------|
| 各社の調査データ | `tmp/researcher/{塾名}.md` |
| 統合構成案 | `tmp/planner/structure.md` |
| 統合レポート | `output/{テーマ名}/reports/{テーマ名}.md` |
| スライド（pptx） | `output/{テーマ名}/slides/{テーマ名}.pptx` |
| スライド画像 | `output/{テーマ名}/images/slide-NN.png` |

レポートは8セクション構成（エグゼクティブサマリー／運営概要比較／コース・指導形態比較／料金体系比較／合格実績比較／地域における認知・評判／差別化要因・ポジショニング／再就職検討への示唆）で、各セクション内で対象学習塾を横断比較する形式になっています。

## 実行例（試走実績）

これまでに以下の2回の試走を実施し、パイプライン全体（researcher並列→planner→writer→reviewer→slide-builder）が正常に動作することを確認しています。

| テーマ名 | 対象エリア | 対象学習塾 | 成果物 |
|---|---|---|---|
| `fukuoka-juku` | 福岡県 | 英進館・全教研・九大進学ゼミ・森田修学館（4社） | `output/fukuoka-juku/` 配下にレポート・スライド・画像 |
| `kyushu-hokubu-8sha` | 福岡・佐賀・長崎・熊本の4県 | 英進館・全教研・九大進学ゼミ・創研学院・能開センター・智翔館・早稲田スクール・熊本ゼミナール（8社） | `output/kyushu-hokubu-8sha/` 配下にレポート・スライド・画像 |

調査の過程で、「創研学院」「能開センター」のように同一ブランド名を複数の無関係な会社が使用しているケースが見つかりました。researcherには対象企業について事前情報を決め打ちで与えず、運営会社・展開エリアをゼロから検証させる運用にしています（ブランド名だけで単一企業と決めつけない）。

## セットアップ

- Claude Code（本リポジトリを開いて `.claude/agents` `.claude/skills` を認識させる）
- Node.js v20以上（`md-to-slides` Skillのスライド生成に使用）
- Python 3 + Playwright（スライド画像レンダリングに使用）

## 実行方法

Claude Code でこのディレクトリを開き、調査したいエリア・学習塾・学年層をプロンプトで指定して依頼してください。PMエージェントが `CLAUDE.md` の定義に従い、フェーズ0（対象確定）→ フェーズ1（並列調査）→ フェーズ2（構成・執筆・レビュー）→ フェーズ3（スライド生成）の順に自動で進行します。

## 由来

本プロジェクトは、侍エンジニアの学習教材で作成したマルチエージェント練習用リポジトリ（企業の市場調査を行うシステム）と、大学入試調査向けに再設計した `university-admissions-research` をベースに、地域展開の学習塾業界向けに調査軸・レポート構成を再設計したものです。
