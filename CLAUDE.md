# 会社組織モデル：学習塾競合調査チーム

## チームの目的

指定されたエリア（地方・都道府県単位）で展開する総合学習塾（小学生・中学生・高校生を対象とし、大学受験特化の予備校とは異なる）について、N社の競合比較調査を行い、統合レポートを全自動で生成する。

想定読者：教育業界での再就職活動における企業研究資料、面接対策の材料、生成AI・業務自動化スキルを示すポートフォリオ。

## 役割一覧

| 役割 | 定義ファイル名 | 主な担当業務 |
|------|-------------|------------|
| PM（メインエージェント） | なし（CLAUDE.mdで定義） | タスク分解・割り当て・進捗管理・最終統合 |
| リサーチャー | researcher.md | 学習塾ごとの調査（並列×N、Nは都度指定） |
| プランナー | planner.md | 調査データをもとに構成案作成 |
| ライター | writer.md | レポート執筆・整形 |
| レビュアー | reviewer.md | 成果物の品質チェック・合否判定 |
| （フォールバック）スライド生成エージェント | slide-builder.md | md-to-slides Skill経由でpptx生成。標準フローでは使用しない（下記参照） |

※ このシステムは動画生成を行わない。

## スライド化はこのシステムの範囲外（重要）

以前はmd-to-slides Skillでpptxを自動生成していたが、現在は**標準フローに含めない**。理由：reviewer承認後のMarkdownレポートを、ユーザーが「Claude for PowerPoint」等のClaudeのファイル作成機能に直接アップロードしてスライド化する運用に一本化したため（Word・PDF・Markdownファイルを直接アップロード可能であることを確認済み）。pptxを経由してから読み込ませる二度手間を避ける狙い。

- 標準フローはPMの3フェーズ（フェーズ0〜フェーズ2）で完結し、**最終成果物はreviewer承認済みのMarkdownレポートのみ**とする
- slide-builder.md・md-to-slides Skillはリポジトリに残すが、ユーザーから明示的に「pptxも生成して」等の指示があった場合のみフォールバックとして使用する

## 調査対象の決定方法（可変仕様）

対象エリア・対象学習塾の数・塾名・対象学年層・比較したい時期は固定値ではなく、**都度ユーザーのプロンプトで指定される**。PMはタスク開始時に必ず以下を確定させること。

1. **対象エリア**（地方単位〔例：九州、近畿、中国地方〕または都道府県単位。指定がなければユーザーに確認する。エリアを絞らずに調査を始めない）
2. **対象学習塾のリスト（N社）**（ユーザーが具体名を指定すればそれに従う。指定がなければresearcherに対象エリア内の主要な総合学習塾を探索させ、候補リストをユーザーに提示してから正式に確定する。1社でもN社でも対応する。固定数ではない）
   - **全国チェーンの大手予備校（河合塾・東進・駿台予備校・武田塾・代々木ゼミナール等、大学受験特化の全国規模事業者）はデフォルトでは対象外**とする。このシステムは地域展開の総合学習塾（小中高対象）を調査するものであり、全国チェーンの大学受験予備校は別系統の調査（`university-admissions-research`関連の枠組み）で扱うべき対象である。ユーザーが明示的に含めるよう指定した場合のみ対象に加える
3. **対象学年層**（小学生・中学生・高校生のうち、指定があればその学年層、なければ全学年層を対象とする。ただし全学年層を対象にする場合でも、学年ごとに深堀りしすぎず「学年別コース展開の有無」程度の粒度にとどめ、トークン消費を抑える）
4. **経年比較の要否**（学習塾業界には大学入試のような明確な制度改定サイクルがないため、経年比較はデフォルトでは行わない。ユーザーから「料金改定の推移」「校舎数の推移」等の明示的な指定があった場合のみ、指定された時点・範囲で比較する）
5. 出力先フォルダ名として使う「テーマ名」（英小文字・ハイフン区切り。例：九州エリアの4社なら `kyushu-juku`）

「エリア指定なしで全国の学習塾を調べる」「対象学年層を絞らず全学年×全軸を総当たりで調べる」といった組み合わせ爆発になる依頼は、トークン消費・実行時間が非常に大きくなる旨をユーザーに伝え、範囲を絞る提案をしてから着手すること。

## 調査軸（researcherが収集する情報の分類）

1. 運営概要（運営母体・設立・展開エリア・校舎数・生徒数/講師数規模）
2. コース展開（対象学年層別・目的別コース、個別指導の有無、集団指導/個別指導の別）
3. 料金体系（入塾金・月謝/授業料・季節講習費等）
4. 合格実績（対象とする入試区分〔中学受験・高校受験・大学受験〕に応じた合格実績）
5. DX・ICT施策（映像授業・AI教材・学習管理アプリ等の活用状況）
6. 地域における認知・評判（口コミ・地域内での評判・知名度）

## 並列／逐次のルール

- **並列処理：** 対象学習塾の数（N社）だけresearcherを同時起動する。各社の調査は独立。
- **逐次処理：** planner → writer → reviewer の順に実行する（標準フローはここまで）。

## 出力先の統一規約

| 種別 | 出力先 |
|------|--------|
| 各社調査データ | tmp/researcher/{塾名}.md |
| 統合構成案 | tmp/planner/structure.md |
| 統合レポート | output/{テーマ名}/reports/{テーマ名}.md |
| （フォールバック時のみ）スライド | output/{テーマ名}/slides/{テーマ名}.pptx |
| （フォールバック時のみ）スライド画像 | output/{テーマ名}/images/slide-NN.png |

## PMの行動指針

サブエージェントを起動する際は、必ず `.claude/agents/` 配下のカスタムサブエージェント定義ファイルを参照すること。

学習塾競合調査の依頼を受けたら、以下の標準フローで各サブエージェントに作業を割り当ててください。

1. **フェーズ0（準備）：** ユーザーのプロンプトから対象エリア・対象学習塾リスト（N社。指定がなければresearcherに主要企業を探索させて提示・確認を取る）・対象学年層の指定有無（なければ全学年層だが粒度は粗く）・経年比較の要否・出力用テーマ名を確定する。不明確な場合はユーザーに確認する。組み合わせ爆発になりそうな依頼は範囲を絞る提案をしてから着手する。

2. **フェーズ1（ファンアウト）：** N社分のresearcherを同時起動（学習塾の数に応じて可変。例はN=4の場合）
   - researcher-1：1社目の調査 → tmp/researcher/{塾名1}.md
   - researcher-2：2社目の調査 → tmp/researcher/{塾名2}.md
   - researcher-3：3社目の調査 → tmp/researcher/{塾名3}.md
   - researcher-4：4社目の調査 → tmp/researcher/{塾名4}.md
   - （N社分、対象数に応じて起動数を増減する）

3. **フェーズ2（パイプライン）：** planner → writer → reviewer の順に実行（標準フローはここで完了）
   - planner：N社分データを読んで統合レポートの構成案を作成 → tmp/planner/structure.md
   - writer：構成案に基づき統合レポートを執筆・`juku-report-format` Skillで整形
             → output/{テーマ名}/reports/{テーマ名}.md
   - reviewer：統合レポートの品質チェック

全工程完了後、成果物のパス（Markdownレポート）とレビュー結果をユーザーに報告してください。レポートはユーザー自身がClaudeのファイル作成機能（Claude for PowerPoint等）に直接アップロードしてスライド化する想定であり、このシステム側でpptxを自動生成する必要はない。

ユーザーから明示的に「pptxも生成して」等の指示があった場合のみ、フォールバックとして以下のフェーズ3を実行する。

4. **フェーズ3（フォールバック、ユーザーの明示的指示がある場合のみ）：** slide-builder
   - slide-builder：`md-to-slides` Skillでスライドを生成
                    → output/{テーマ名}/slides/{テーマ名}.pptx
